통합인증 데모 · 기획 개요

하나의 신원, 여러 개의 로그인

PASS 본인인증 · 소셜 로그인 · 아이디/비밀번호 로그인을 한 사람의 계정으로 묶는 서비스입니다. 기획·디자인 관점에서 무엇을 만들고 사용자가 어떤 경험을 하는지 정리했습니다.

이 서비스가 하려는 것

"이 사람은 실제로 누구인가"를 기준으로, 여러 로그인 수단을 한 사람에게 묶습니다.

사용자는 카카오로도, 네이버로도, 아이디/비밀번호로도 로그인할 수 있지만, 그게 모두 같은 한 사람의 계정으로 연결됩니다. 기준이 되는 것은 본인인증(PASS)으로 확인한 실제 신원입니다.

🪪

본인인증이 기준점

PASS로 확인한 실제 신원이 계정의 뿌리가 됩니다.

🔗

여러 로그인 수단

소셜 계정 여러 개 + 아이디/비밀번호 1개를 한 사람에게 연결.

👤

중복 가입 방지

같은 사람이 실수로 계정을 여러 개 만드는 것을 막습니다.

핵심 개념 — 왜 본인인증이 중심인가

소셜 로그인만으로는 "이 사람이 진짜 누구인지" 알 수 없습니다.

카카오 계정과 네이버 계정은 서로 다른 서비스라, 그 자체로는 같은 사람인지 알 방법이 없습니다. 그래서 본인인증으로 확인한 실제 신원을 기준으로 삼습니다. 한 번 본인인증을 하면, 그 사람에게 붙는 소셜·로컬 계정은 모두 "같은 사람"으로 묶입니다.

쉽게 말하면 본인인증 = 계정들을 하나로 꿰는 실입니다. 카카오·네이버·아이디는 실에 꿰이는 구슬이고, 실은 "실제 그 사람"입니다.

사용자 여정

가입부터 재로그인까지, 사용자가 겪는 흐름입니다.

처음 오는 사람 — 가입

1

로그인 수단 선택

카카오·네이버·구글, 또는 아이디/비밀번호 가입 중 선택.

2

본인인증 (PASS)

실제 신원을 한 번 확인합니다. 이 서비스의 첫 관문입니다.

3

가입 완료 & 로그인

선택한 로그인 수단이 그 사람에게 연결되고 바로 로그인됩니다.

이미 가입한 사람 — 재로그인

한 번 연결한 수단으로는 본인인증 없이 바로 로그인됩니다. 카카오로 가입했으면 다음부터는 카카오 버튼만 누르면 끝입니다.

다른 로그인 수단 추가하기

로그인한 상태에서 "네이버도 연결하기", "아이디/비밀번호 설정하기"처럼 계정에 로그인 수단을 더 붙일 수 있습니다. 다음부터는 어느 쪽으로 들어와도 같은 계정입니다.

소셜 로그인으로 처음 들어온 사람은?

"가입 안 하고 소셜 로그인"이라는 상태는 따로 없습니다 — 자연스럽게 가입으로 이어집니다.

예를 들어 처음 온 사람이 "카카오로 로그인"을 누르면, 아직 연결된 적이 없으니 본인인증 화면으로 안내됩니다. 본인인증을 마치면 그 순간 가입이 완료되고, 이후엔 카카오 버튼만으로 바로 로그인됩니다.

본인인증은 딱 한 번 각 로그인 수단은 처음 연결할 때만 본인인증을 거칩니다. 한 번 연결되면 그 다음부터는 매번 인증하지 않습니다.
이미 가입한 사람이 새 소셜로 들어오면 본인인증 결과가 기존에 가입한 그 사람과 동일인으로 확인되면, 새 소셜 계정이 기존 계정에 안전하게 연결됩니다. (실제 본인 확인이 끝난 상태라 다른 사람 계정에 잘못 붙을 걱정이 없습니다.)

사용자 입장에서 정리

완전 신규가 카카오 클릭
카카오 인증 → 본인인증 1회 → 가입 완료 & 로그인
아이디로 가입했던 사람이 카카오 클릭
카카오 인증 → 본인인증 1회 → "기존 계정에 연결됨" → 로그인
카카오 연결이 끝난 사람이 카카오 클릭
카카오 인증 → 바로 로그인 (본인인증 없음)

데모 단계에서의 본인인증

실제 PASS 연동은 계약이 필요해, 데모에서는 "테스트용 본인인증"으로 대체합니다.

실제 PASS 본인인증은 인증 대행사와 계약을 해야 사용할 수 있습니다. 그래서 데모 단계에서는 실제 PASS 대신 테스트용 본인인증 화면을 사용합니다. 사용자가 이름·생년월일 등 정보를 입력하면, 실제 인증을 한 것처럼 다음 단계로 넘어갑니다.

테스트용이라 이렇게 동작합니다
  • 같은 정보로 다시 인증하면 같은 사람으로 인식 → "이미 가입됨 / 기존 계정 연결" 흐름을 확인할 수 있습니다.
  • 정보를 다르게 넣으면 다른 사람으로 인식 → 신규 가입 흐름을 확인할 수 있습니다.
나중에 실제 PASS로 바꿀 때 화면 흐름과 사용자 경험은 그대로 두고, 본인인증 부분만 실제 PASS로 교체하도록 설계합니다. 기획·디자인 산출물은 그대로 쓸 수 있습니다.

함께 정해야 할 것

화면 설계로 넘어가기 전에 기획 관점에서 결정이 필요한 항목입니다.

가입할 때 본인인증을 언제 요구할까?
"본인인증을 먼저 하고 로그인 수단을 등록" 방식으로 통일할지. (신원 기준이 명확해져 흐름이 깔끔해집니다.)
기존 계정에 새 소셜을 연결할 때, 확인 화면을 보여줄까?
본인인증으로 동일인이 확인되면 말없이 자동 연결할지, 아니면 "○○○님 계정에 카카오를 연결할까요?" 확인 화면을 한 번 보여줄지. → 디자인이 필요한 화면이 달라집니다.
테스트용 본인인증 화면은 어디에?
실제 PASS처럼 별도 인증창으로 이동하는 방식과, 서비스 안의 간단한 입력 팝업 방식 중 선택. (앞의 방식이 실제 PASS 경험에 더 가깝습니다.)
