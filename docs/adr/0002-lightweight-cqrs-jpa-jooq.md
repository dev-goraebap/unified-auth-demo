# ADR-0002. 경량 CQRS와 데이터 접근 — 명령 JPA / 조회 jOOQ(동적 타입)

- 상태: 수용(Accepted)
- 일자: 2026-07-19

## 맥락

인증 도메인은 **쓰기(명령)** 와 **읽기(조회)** 의 성격이 뚜렷이 다르다.

- 명령: 사용자·계정 생성, 소셜 연결, 로컬 자격증명 설정 등 — 도메인 불변식을 지켜야 한다
  (예: "한 소셜 계정은 한 사용자에만", "로컬 계정은 사용자당 하나").
- 조회: 관리 화면·마이페이지처럼 여러 테이블을 조인해 화면에 맞춘 데이터를 뽑는다 —
  도메인 규칙보다 화면 요구에 의존적이다.

이 둘을 하나의 모델·기술로 억지로 다루면, 조회 편의를 위해 도메인 모델이 오염되거나
반대로 도메인 보호를 위해 조회가 불편해진다.

## 결정

**경량 CQRS**를 적용한다. 요구사항을 명령과 조회로 나누고, 각기 다른 데이터 접근 기술을 쓴다.

### 명령 — Spring Data JPA

- 도메인 엔티티에 JPA 애너테이션(`@Entity`, `@Id` 등)을 **직접** 허용한다.
  ORM 엔티티와 도메인 엔티티를 분리하지 않는다(데모 규모에서 분리는 과함).
- 도메인 저장소 인터페이스는 `JpaRepository`를 **직접 상속**한다. 저장소 구현체를
  손으로 만들지 않는다.
- 프레임워크 완전 독립은 포기하되, **자가 검증**은 유지한다 —
  `protected` 기본 생성자 + 정적 팩토리로 생성을 통제해, 엔티티가 스스로 불변식을 지킨다.

### 조회 — jOOQ (동적 타입, 코드젠 없음)

- 화면 조회는 JPA로 억지로 풀지 않고 **jOOQ로 SQL을 직접** 작성한다.
- **동적 타입(dynamic)** 으로 쓴다 — jOOQ의 정적 코드 생성(코드젠)을 **사용하지 않는다.**
  스키마로부터 타입 세이프한 클래스를 생성하는 대신, `DSL.field(...)` / `DSL.table(...)`
  등 문자열 기반 API로 쿼리를 조립한다.
- 조회는 외부 결과를 화면 DTO(record)로 매핑해 반환한다.

### 채택하지 않은 것

- **MyBatis**: 채택하지 않는다. 조회 기술은 jOOQ로 단일화한다.
- **jOOQ 정적 코드젠**: 채택하지 않는다(아래 "결과" 참고).

## 결과

- **장점**
  - 명령은 JPA로 도메인 불변식을 안전하게 지키고, 조회는 jOOQ로 화면에 최적화된 SQL을
    자유롭게 쓴다 — 각자 잘하는 일을 한다.
  - 코드젠이 없어 스키마가 바뀌어도 빌드 파이프라인에 생성 단계가 끼지 않는다.
    데모에서 반복이 빠르다.
- **비용 / 트레이드오프**
  - 동적 jOOQ는 **컴파일 타임 타입 안전성을 포기**한다. 컬럼명 오타·타입 불일치가
    런타임에 드러난다. → 데모 규모에서 수용하며, **조회는 통합 테스트로 SQL·스키마
    정합성을 확인**해 이 위험을 상쇄한다.
  - 조회가 도메인 모델을 우회하므로, 조회 코드가 도메인 상태를 바꾸는 일이 없도록
    규율이 필요하다(조회는 읽기 전용).
- **연관**: DB는 [ADR-0003](0003-postgresql-local-docker-demo-db.md) 참고. 스키마 관리
  도구(Flyway/Liquibase 등) 도입 여부는 ADR-0003에서 다룬다.
